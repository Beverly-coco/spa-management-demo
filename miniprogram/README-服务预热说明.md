# 云服务预热机制说明

## 问题背景

云托管服务存在"冷启动"现象：

- 当服务长时间未被访问时，会进入休眠状态
- 首次请求会触发服务启动，耗时较长（通常 10-30 秒）
- 导致用户首次点击登录时出现超时错误，需要重试 2-3 次才能成功

## 解决方案

实现了**服务预热机制**，在小程序启动时就提前唤醒云服务：

### 1. 小程序启动时预热 (app.js)

在 `onLaunch` 生命周期中：

- 云环境初始化完成后，立即发送预热请求到 `/health` 端点
- 如果首次预热失败，会在 3 秒后自动重试
- 最多重试 2 次，每次间隔 5 秒
- 预热过程在后台静默进行，不影响用户使用

```javascript
// 预热流程
小程序启动
  ↓
云环境初始化
  ↓
立即发送预热请求 (/health)
  ↓
失败 → 3秒后重试 (/ping)
  ↓
失败 → 5秒后再次重试 (/ping)
  ↓
成功 → 设置 serviceWarmedUp = true
```

### 2. 登录页面二次预热 (login.js)

在登录页面 `onLoad` 时：

- 检查全局状态 `serviceWarmedUp`
- 如果服务尚未预热成功，再发起一次预热请求
- 确保用户点击登录前，服务已经启动

### 3. 优化的错误提示

根据预热状态提供不同的错误提示：

- 如果服务尚未预热且超时：提示"云服务首次启动需要一些时间，请稍等片刻后重试登录"
- 如果服务已预热但仍超时：提示"请求超时，请重试"
- 其他错误：提供相应的错误信息

## 技术实现

### app.js 修改

1. **warmupCloudService()** - 主预热函数

   - 调用 `/health` 端点
   - 超时时间：10 秒
   - 失败后触发重试机制

2. **retryWarmup(retryCount)** - 重试函数

   - 调用 `/ping` 端点
   - 超时时间：15 秒
   - 最多重试 2 次
   - 使用 `setTimeout` 控制重试间隔

3. **globalData.serviceWarmedUp** - 状态标志
   - 初始值：false
   - 预热成功后设为 true
   - 用于判断服务是否就绪

### login.js 修改

1. **warmupServiceForLogin()** - 登录页预热

   - 在 `onLoad` 时检查并触发
   - 静默失败，不打扰用户

2. **优化的错误处理**
   - 根据 `serviceWarmedUp` 状态提供不同提示
   - 更友好的错误信息

## 时间线示例

```
0秒   - 用户打开小程序
0.1秒 - 云环境初始化
0.2秒 - 发送第1次预热请求 → 触发云服务启动
      - 用户看到闪屏页/首页

      - 用户进入登录页
      - 登录页检测到服务已预热

      - 请求立即成功！✅
```

## 注意事项

1. **预热不影响应用启动**

   - 预热在后台异步进行
   - 即使预热失败，用户仍可正常使用
   - 服务会在真正请求时启动

2. **超时时间设置**

   - 首次预热：10 秒
   - 重试预热：15 秒
   - 登录请求：60 秒（app.js 中的 callCloudApi）

3. **后端健康检查端点**

   - `/health` - 主健康检查端点
   - `/ping` - 备用健康检查端点
   - `/healthz` - Kubernetes 风格健康检查
   - 这些端点响应快速，不涉及数据库操作

4. **日志输出**
   - 所有预热过程都有详细日志
   - 便于调试和监控
   - 生产环境可通过日志分析预热效果

## 进一步优化建议

如果仍有冷启动问题，可以考虑：

1. **云托管配置**

   - 调整最小实例数为 1（保持至少 1 个实例常驻）
   - 设置更长的闲置超时时间
   - 启用预留实例

2. **定时触发器**

   - 创建云函数定时触发器
   - 每 5-10 分钟调用一次健康检查
   - 保持服务持续活跃

3. **CDN 预热**
   - 对于静态资源，使用 CDN 预热
   - 减少首次加载时间

## 测试方法

1. 完全关闭小程序（从后台清除）
2. 等待 5 分钟让云服务休眠
3. 重新打开小程序
4. 查看控制台日志，确认预热请求
5. 进入登录页，点击登录
6. 应该首次点击就能成功登录
